# ♠ ポーカーチップカウンター

テキサスホールデムを実際のカードで遊びながら、チップ(架空のお金)の管理だけをアプリで行うためのツールです。各プレイヤーが自分のスマホからルームに参加し、チップの状態がリアルタイムで同期されます。

- カードは実物のトランプを使います(アプリはカードを配りません)
- ベット / コール / レイズ / フォールド / オールインをアプリ上で操作
- ポットとサイドポットを自動計算
- ブラインド(SB/BB)の自動徴収、ディーラーボタンの自動移動
- ショーダウンではホストが勝者を選択してポットを分配(引き分け分割対応)

## 仕組み

- **GitHub Pages** — アプリ本体(静的サイト)の配信
- **Firebase Realtime Database** — ルーム状態のリアルタイム同期(無料枠で十分)

## セットアップ手順

### 1. Firebase プロジェクトを作成する

1. [Firebase コンソール](https://console.firebase.google.com/) にアクセスし、Google アカウントでログイン
2. 「プロジェクトを追加」→ 任意の名前(例: `poker-counter`)で作成(Google アナリティクスは無効でOK)
3. 左メニューの「構築」→「Realtime Database」→「データベースを作成」
   - ロケーションは `asia-southeast1` など近い場所を選択
   - セキュリティルールは「**テストモード**」を選択
4. 「ルール」タブで以下に書き換えて公開(テストモードは30日で失効するため):

```json
{
  "rules": {
    "rooms": {
      ".read": true,
      ".write": true
    }
  }
}
```

> ⚠️ このルールは誰でも読み書きできる設定です。友人内で遊ぶ用途を想定しています。

### 2. ウェブアプリの構成情報を取得する

1. プロジェクトの設定(歯車アイコン)→「全般」→「マイアプリ」→ ウェブアプリ(`</>` アイコン)を追加
2. 表示される `firebaseConfig` の値を、このリポジトリの [`src/firebaseConfig.ts`](src/firebaseConfig.ts) に貼り付ける
   - `databaseURL` が含まれていない場合は、Realtime Database の画面上部に表示される URL(`https://xxx.firebasedatabase.app`)を追記

### 3. GitHub Pages を有効にする

1. リポジトリの Settings → Pages → 「Source」を「**GitHub Actions**」に設定
2. `main` ブランチにプッシュ(マージ)すると自動でビルド・デプロイされます
3. 公開された URL を友人に共有すれば、みんなのスマホからアクセスできます

## 遊び方

1. ホストが「ルームを作る」で初期チップとブラインドを設定してルームを作成
2. 表示された6桁のルームコードを友人に共有し、各自「ルームに参加」
3. ホストが「ゲーム開始」→ ブラインドが自動で徴収されます
4. 実物のカードを配り、画面の指示に従って各自ベットを操作
5. ショーダウンになったら手札を見せ合い、ホストが勝者をタップしてポットを分配
6. 「次のハンドへ」でディーラーボタンが移動し、次のハンドが始まります

## 開発

```bash
npm install
npm run dev    # 開発サーバー
npm test       # ポーカーロジックのユニットテスト
npm run build  # 型チェック + ビルド
```

### サウンドと見た目

- 効果音とBGMは Web Audio API でその場で合成しています(音声ファイル不要・著作権フリー)
- 画面右上の 🎵 / 🔊 ボタンでいつでも ON/OFF できます(設定は端末ごとに保存)
- サウンドとリッチUIは独立したコミットになっているため、問題があれば
  `git revert <コミットSHA>` でそれぞれ個別に取り消せます

### 実装済みのポーカールール

- ヘッズアップ(2人)ではディーラーが SB、プリフロップはディーラーから行動
- BB のオプション(全員コールでも BB はチェック / レイズを選べる)
- 最低レイズ幅 = 直前のフルレイズの幅(最初は BB)
- フルレイズ未満のオールインでは、行動済みプレイヤーのレイズ権は復活しない(TDA ルール)
- チップ不足のコール / ブラインドは自動的にオールイン
- サイドポットの自動計算と、コールされなかった超過ベットの自動返却
- スプリットポットの端数はディーラーの左隣に近い勝者から受け取る
- ディーラーボタンはチップを持つ次のプレイヤーへ移動
- バーストしたプレイヤーへのリバイ(ホスト操作)、ハンド間のブラインド変更
